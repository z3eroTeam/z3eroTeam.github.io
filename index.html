python3多进程

发表于 2018-08-22 | 分类于 Python3

由于这两天自己负责的数据处理模块，都涉及到IO密集的数据读写，为了能够加快处理速度和调试速度，就想着使用Python的多进程，顺便借此机会，系统地学习一下Python的多进程。

多进程主要由以下几种方式：

  1. fork（仅支持Unix/Linux系统）
  2. multiprocessing（跨平台多进程）
    • Pool
    • Process

依次来看下这几种方式创建和实现多进程。

1. fork

fork其实是操作系统提供的API，操作系统从当前进程（父进程）创建（说“复制”可能更恰当）一个子进程。fork函数会从父进程和子进程分别都返回一次。

第2,3行：结合两个输出提示一起看，不难理解os.fork()返回的pid=18841是其新复制出来的子进程的进程号。此时，正在执行的是父进程。

第4,5行：此时输出pid=0，且正在执行的是子进程。

可以理解成，从os.fork()往后，我们的进程复制了一份（包括fork后面的代码），变成了两个进程。

两个进程同时在执行，只不过在子进程中，fork返回的永远是0，父进程中返回的时子进程的ID。

因此可以这样通过简单判断的返回的结果，来区别子进程和父进程不同的工作。

但是这种方法不好的地方，主要存在于限制了代码运行平台，对于python这种跨平台的语言来说，显然是不合适的。

解决Apache的因为GLIBC_PRIVATE not defined in file libc.so.6，而Graceful restart failed

发表于 2018-08-22 | 分类于 解决方案

问题发现

在访问系统程序时，发现无法加载出web界面。同时80端口的apache页面也无法访问。

解决过程

  1. 查看apache日志文件
    发现在error.log.*的日志中，出现Apache graceful restart failed的错误。其中涉及到三个so文件[libphp7.0.so,libresolv.so.2,libc.so.6]
  2. 确认是否存在这三个文件
    经过查找确认，发现三个文件都存在。
  3. 查找资料，了解apache graceful restart的机制
    好像是因为，apache会定时进行log rotation，然后会进行像软重启的机制。
  4. 查看Apache的配置文件
    在配置文件里面，发现log_rotation默认设置为daily。（然而实际上这对解决问题，也没啥帮助）
  5. 查找现有解决方案
    因为GLIBC_PRIVATE和libc.so.6的错误，基本上多少都与文件丢失，或者在某个指定目录下找不到libc.so.6文件有关

    假设性解决

    将/lib/x86_64-Linux-gnu/libc.so.6软连接到/lib，/lib64和/libc.so6目录下。

————
等待两天后，发现web页面能够还是能够正常访问。

虽然没搞懂原理，但是就先暂时当作一种解决方案吧。

顺便提一下，libc.so.6是Linux下重要的C的动态连接库，里面包含了很多Linux的基本命令，所以没事就别瞎搞它。

web应用无法读取Hbase数据

发表于 2018-07-31 | 分类于 实体机部署问题

问题描述

在部署山东实体机时，发现web应用无法读取到Hbase中的数据，导致DDOS模块中流量还原不能正常显示以及域名历史记录添加新的域名后不能正常显示。编写了一个小的java程序在部署web服务器上，来读取Hbase集群中的数据，发现程序不能正常读取数据，但是也不会马上报错。

解决方法

查询各种博客无果，但是通过分析等待几十分钟后的异常结果显示发现，程序在查询数据时，用到了Hbase中的主机名字，然而web服务器上没有对应的匹配。所以我将Hbase集群上的 /etc/hosts 文件中的各个机器IP以及对应的主机名（hostname）复制粘贴到了web服务器的/etc/hosts文件中，问题解决。
